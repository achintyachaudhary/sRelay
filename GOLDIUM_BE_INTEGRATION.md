# Goldium — Stock Market Backend Integration Guide

> **Purpose:** This document is the single source of truth for your **main stock market backend** to push data into **Goldium** (iOS app) via **StockRelay** on Render.
>
> Feed this file to Cursor when implementing the stock market backend.

---

## System Architecture

```
┌─────────────────────┐      POST /api/v1/ingest       ┌──────────────────────┐      WebSocket       ┌─────────────────┐
│  Stock Market BE    │  ─────────────────────────────►  │  StockRelay (Render) │  ──────────────────►  │  Goldium iOS    │
│  (your main server) │      https://srelay.onrender.com│  In-memory relay     │  wss://srelay.../ws  │  Twitter feed   │
└─────────────────────┘                                  └──────────────────────┘                      └─────────────────┘
```

| Component | Role |
|-----------|------|
| **Stock Market BE** | Generates triggers & EOD summaries from your analytics |
| **StockRelay** | Public relay — stores messages in memory, fans out to iOS clients |
| **Goldium iOS** | Read-only Twitter-like feed; stores history locally |

**Relay URL:** `https://srelay.onrender.com`  
**Ingest endpoint:** `POST https://srelay.onrender.com/api/v1/ingest`  
**No auth required for now** (public ingest).

---

## Message Contract

Every message **must** have:

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `id` | string | Recommended | Unique ID for dedup. Auto-generated by relay if omitted. |
| `type` | string | **Yes** | `"trigger"` or `"eod_summary"` |
| `timestamp` | string (ISO 8601) | Recommended | e.g. `2026-06-08T15:30:00Z`. Auto-generated if omitted. |

Goldium **only renders** `trigger` and `eod_summary` with dedicated cards. Other types fall back to a generic card.

## Type 1: `trigger` — Price Alert

Sent when a stock crosses a threshold (e.g. moved above ₹1000 or below ₹650).

### JSON Schema

```json
{
  "id": "trigger_reliance_20260608_001",
  "type": "trigger",
  "timestamp": "2026-06-08T15:30:00+05:30",
  "stock": {
    "symbol": "RELIANCE",
    "name": "Reliance Industries",
    "current_price": 2845.50,
    "change_percent_today": 1.25,
    "industry": "Oil & Gas"
  },
  "alert": {
    "direction": "above",
    "threshold": 2800,
    "label": "RELIANCE moved above ₹2800"
  },
  "nifty": {
    "value": 23542.30,
    "change_percent_today": 0.38
  }
}
```

### Field Reference

#### `stock` object

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `symbol` | string | Yes | Ticker symbol |
| `name` | string | Yes | Full company name |
| `current_price` | number | Yes | Current stock price |
| `change_percent_today` | number | Yes | Today's % change (e.g. `1.25` or `-0.85`) |
| `industry` | string | Yes | Sector/industry label |

#### `alert` object

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `direction` | string | Yes | `"above"` \| `"below"` \| `"up"` \| `"down"` |
| `threshold` | number | No | Price level that was crossed |
| `label` | string | Yes | Human-readable alert text shown on card |

#### `nifty` object

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `value` | number | Yes | Current NIFTY 50 value |
| `change_percent_today` | number | Yes | NIFTY today's % change |

### curl Example

```bash
curl -X POST https://srelay.onrender.com/api/v1/ingest \
  -H "Content-Type: application/json" \
  -d '{
    "type": "trigger",
    "stock": {
      "symbol": "RELIANCE",
      "name": "Reliance Industries",
      "current_price": 2845.50,
      "change_percent_today": 1.25,
      "industry": "Oil & Gas"
    },
    "alert": {
      "direction": "above",
      "threshold": 2800,
      "label": "RELIANCE moved above ₹2800"
    },
    "nifty": {
      "value": 23542.30,
      "change_percent_today": 0.38
    }
  }'
```

## Type 2: `eod_summary` — End of Day Portfolio Summary

Sent once at market close with portfolio performance and per-stock breakdown.

### JSON Schema

```json
{
  "id": "eod_2026-06-08",
  "type": "eod_summary",
  "timestamp": "2026-06-08T15:45:00+05:30",
  "date": "2026-06-08",
  "portfolio": {
    "invested": 850000,
    "current_value": 872450,
    "today_pnl": 4820,
    "today_pnl_percent": 0.56
  },
  "stocks": [
    {
      "symbol": "RELIANCE",
      "name": "Reliance Industries",
      "invested": 200000,
      "current_value": 208500,
      "today_pnl": 2100,
      "today_pnl_percent": 1.02,
      "status": "profit"
    },
    {
      "symbol": "TCS",
      "name": "Tata Consultancy Services",
      "invested": 150000,
      "current_value": 147200,
      "today_pnl": -980,
      "today_pnl_percent": -0.66,
      "status": "loss"
    }
  ]
}
```

### Field Reference

#### `portfolio` object

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `invested` | number | Yes | Total capital invested |
| `current_value` | number | Yes | Total current portfolio value |
| `today_pnl` | number | Yes | Today's profit/loss in ₹ |
| `today_pnl_percent` | number | Yes | Today's P&L as % |

#### `stocks[]` array (one entry per holding)

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `symbol` | string | Yes | Ticker |
| `name` | string | Yes | Company name |
| `invested` | number | Yes | Amount invested in this stock |
| `current_value` | number | Yes | Current value of holding |
| `today_pnl` | number | Yes | Today's P&L for this stock |
| `today_pnl_percent` | number | Yes | Today's % change for this stock |
| `status` | string | Yes | `"profit"` \| `"loss"` \| `"flat"` |

## Batch Ingest

Push multiple messages in one request:

```bash
curl -X POST https://srelay.onrender.com/api/v1/ingest \
  -H "Content-Type: application/json" \
  -d '{
    "messages": [
      { "type": "trigger", "stock": { "symbol": "INFY", "name": "Infosys", "current_price": 1520, "change_percent_today": -0.5, "industry": "IT" }, "alert": { "direction": "below", "threshold": 1550, "label": "INFY dropped below ₹1550" }, "nifty": { "value": 23500, "change_percent_today": -0.2 } },
      { "type": "eod_summary", "date": "2026-06-08", "portfolio": { "invested": 500000, "current_value": 510000, "today_pnl": 2000, "today_pnl_percent": 0.4 }, "stocks": [] }
    ]
  }'
```

```json
      "stock": { "symbol": "RELIANCE", "..." : "..." }
    }
  ]

```

Relay immediately broadcasts accepted messages to all connected Goldium clients over WebSocket.

## When to Push What

| Event | Type | Frequency |
|-------|------|-----------|
| Stock crosses price threshold | `trigger` | Real-time, as events occur |
| Market close portfolio summary | `eod_summary` | Once per day after market close |

---

## Implementation Checklist for Stock Market BE

1. **On trigger event** → build `trigger` JSON → `POST /api/v1/ingest`
2. **At market close** → aggregate portfolio → build `eod_summary` JSON → `POST /api/v1/ingest`
3. Use **unique `id`** per message (e.g. `trigger_{symbol}_{date}_{seq}`)
4. Always include **`change_percent_today`** for both stock and NIFTY
5. Use **`status: "profit"` / `"loss"`** based on `today_pnl` sign for EOD stocks
6. Handle HTTP `201` as success; log `4xx`/`5xx` errors

### Minimal Python Example

```python
import requests

RELAY_URL = "https://srelay.onrender.com/api/v1/ingest"

def push_trigger(symbol, name, price, change_pct, industry, direction, threshold, label, nifty_val, nifty_chg):
    payload = {
        "type": "trigger",
        "stock": {
            "symbol": symbol,
            "name": name,
            "current_price": price,
            "change_percent_today": change_pct,
            "industry": industry,
        },
        "alert": {
            "direction": direction,
            "threshold": threshold,
            "label": label,
        },
        "nifty": {
            "value": nifty_val,
            "change_percent_today": nifty_chg,
        },
    }
    resp = requests.post(RELAY_URL, json=payload, timeout=10)
    resp.raise_for_status()
    return resp.json()

def push_eod_summary(date, portfolio, stocks):
    payload = {
        "type": "eod_summary",
        "date": date,
        "portfolio": portfolio,
        "stocks": stocks,
    }
    resp = requests.post(RELAY_URL, json=payload, timeout=10)
    resp.raise_for_status()
    return resp.json()
```

## Goldium iOS App Behavior

| Behavior | Detail |
|----------|--------|
| Feed layout | Twitter-style timeline, newest on top |
| Connection | WebSocket primary (`wss://srelay.onrender.com/ws/messages`) |
| Keepalive | `GET /health` every 7 minutes |
| Local storage | All messages saved on device (`messages.json`, max 500) |
| Themes | Light (default) and Dark |
| Notifications | Optional local alerts on new messages |

## Limitations

- StockRelay free tier: in-memory only, messages lost on server restart
- iOS local storage is the durable archive
- Ingest is public (no API key) — add auth before production
- Render spins down after 15 min idle; Goldium keepalive prevents this when app is active
